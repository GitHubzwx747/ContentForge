## 输出要求
只输出严格 JSON，不要输出 Markdown、解释、标题或额外文字。

JSON 最外层 key 必须使用实际平台名称，且只能包含 {platforms} 中出现的平台。

格式如下：

{
  "实际平台名称": {
    "angle": "适合该平台的具体选题角度",
    "audience": "目标受众描述，包括人群特征、关注点或潜在情绪",
    "structure": {
      "hook": "开头策略，用于快速吸引注意力",
      "body": "主体结构，说明内容展开逻辑",
      "cta": "结尾引导，符合平台互动习惯"
    },
    "emotion_hook": "情绪共鸣点，需贴合热点画像中的公众情绪"
  }
}

## 策略要求
- 必须为每个目标平台生成一个策略卡。
- angle 要具体到可直接进入创作，避免“结合热点进行分析”这类空话。
- audience 要说明目标人群和他们关心什么。
- hook 可使用冲突、利益点、反常识、悬念、情绪共鸣等方式。
- body 要说明内容如何展开，可包含背景、观点、案例、数据、对比、风险或解决方案。
- cta 要自然，符合平台互动习惯，避免生硬营销。
- 如果平台特性参考缺失，可基于常见平台生态谨慎推断，但不要编造具体数据。
